# Task 기반 개발 규칙

## 적용 시점

아래 요청이 들어오면 반드시 `/task` 스킬을 통해 task 파일을 생성한다.

- 새로운 기능 구현 (feat)
- 버그 수정 (fix)
- 리팩토링 (refactor)
- 3단계 이상의 작업이 필요한 모든 요청

---

## 파일 네이밍

요청의 큰 단위 작업명을 한국어 그대로 사용하되, 공백은 하이픈으로 변환한다.

```
docs/tasks/{작업명}.md

예)
"엑셀 생성 API 만들어줘"      → docs/tasks/엑셀-생성-API.md
"job 상태 조회 구현해줘"      → docs/tasks/job-상태-조회.md
"주문 목록 polling 붙여줘"    → docs/tasks/주문-목록-polling.md
```

---

## 파일 형식

```markdown
# {작업명}

> 생성일: {YYYY-MM-DD}
> 브랜치: {현재 브랜치명}

## Tasks

- [ ] {세부 작업 1}
- [ ] {세부 작업 2}
- [ ] {세부 작업 3}
- [ ] 테스트 코드 작성
- [ ] 빌드 확인
```

---

## 상태 표기

| 상태   | 파일 표기           | TodoWrite   |
| ------ | ------------------- | ----------- |
| 시작 전 | `- [ ]`            | `pending`   |
| 작업 중 | `- [ ] ~~진행중~~` | `in_progress` |
| 완료   | `- [x]`             | `completed` |

동시에 `in_progress` 항목은 반드시 1개만 유지한다.
