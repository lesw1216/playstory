# PlayStory 사전과제 설계

## 1. 요구사항 분석

### 1-1. 기능 요구사항

- 사용자는 데이터 요청 목록 화면에서 엑셀 생성 요청 목록을 조회할 수 있다.
- 사용자는 데이터 요청 목록 화면에서 엑셀 생성 요청을 할 수 있다.
- 사용자는 엑셀 생성 요청 후 해당 job의 상태를 확인할 수 있다.
- 프론트엔드는 job_id, 요청일시, 시작시간, 완료시간, 상태, 파일경로를 테이블 형태로 표시한다.

### 1-2. 백엔드 요구사항

- 엑셀 생성 요청을 받으면 서버는 즉시 응답을 반환한다.
- 실제 엑셀 생성은 백그라운드에서 비동기로 처리한다.
- 엑셀 생성 요청은 job 단위로 관리한다.
- job 상태는 PENDING, PROCESSING, DONE, FAILED로 관리한다.
- job 요청 시간, 시작 시간, 완료 시간을 DB에 기록한다.
- 엑셀 생성 완료 후 파일 저장 경로를 DB에 기록한다.
- 엑셀 생성 실패 시 FAILED 상태로 변경하고 실패 원인을 기록한다.

### 1-3. 데이터 요구사항

- 주문 데이터 테이블에는 최소 10만 건의 더미 데이터가 자동 삽입되어야 한다.
- 주문 데이터는 id, user_name, product_name, category, amount, status, order_date 컬럼을 가진다.
- status는 confirmed, cancelled, pending 값을 가진다.

### 1-4. 실행 환경 요구사항

- Docker Compose를 사용한다.
- docker compose up --build 한 번으로 전체 서비스가 실행되어야 한다.
- 외부 계정 또는 외부 서비스에 의존하지 않는다.
- 엑셀 파일은 컨테이너 로컬 볼륨에 저장한다.
- Spring Boot 백엔드는 단일 컨테이너로 실행한다.
- Spring Boot 컨테이너는 CPU 0.5, Memory 1G 제한을 따른다.

## 2. 핵심 설계 방향

엑셀 생성 요청은 즉시 처리 결과를 반환할 수 없는 작업이므로, 요청 자체를 하나의 job으로 저장한다.

클라이언트가 엑셀 생성 요청을 보내면 서버는 excel_export_job 테이블에 PENDING 상태의 job을 생성하고, job_id를 즉시 응답한다. 이후 실제 엑셀 생성은 별도 worker 스레드에서 수행한다.

worker가 작업을 시작하면 job 상태를 PROCESSING으로 변경하고 started_at을 기록한다. 엑셀 파일 생성에 성공하면 DONE 상태로 변경하고 completed_at과 file_path를 기록한다. 작업 중 예외가 발생하면 FAILED 상태로 변경하고 completed_at과 error_message를 기록한다.

---

## 3. 상태 흐름

PENDING → PROCESSING → DONE  
PENDING → PROCESSING → FAILED

PENDING은 job이 생성되었지만 worker가 아직 실제 작업을 시작하지 않은 상태다.  
PROCESSING은 worker가 실제 엑셀 파일을 생성 중인 상태다.  
DONE은 엑셀 파일 생성과 저장이 완료된 상태다.  
FAILED는 엑셀 생성 중 예외가 발생하여 실패한 상태다.

---

## 4. 프론트 상태 확인 방식

상태 확인은 Polling 방식을 사용한다.

데이터 요청 목록 화면은 최신 job 목록을 표시해야 하므로, 프론트엔드는 일정 주기로 GET /api/excel-jobs?page=0&size=10 API를 호출한다.

목록 안에 PENDING 또는 PROCESSING 상태의 job이 하나라도 있으면 polling을 유지하고, 모든 job이 DONE 또는 FAILED 상태이면 polling을 중지한다.

화면 진입 시 조회는 1회 조회가 기본이다. 단, 이전에 요청한 작업이 아직 진행 중 이라면 polling을 진행한다.

SSE는 더 실시간성이 높지만, 현재 요구사항은 단순한 job 상태 확인이므로 구현 복잡도와 리소스 점유를 고려해 polling을 선택한다.

## 5. API 설계 초안

**POST** /api/excel-jobs

- 엑셀 생성 job을 생성한다.
- 즉시 job_id와 초기 상태를 응답한다.

**GET** /api/excel-jobs?page=0&size=10

- 최신순으로 job 목록을 조회한다.

**GET** /api/excel-jobs/{jobId}

- 특정 job의 상태를 조회한다.
